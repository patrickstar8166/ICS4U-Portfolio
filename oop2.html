Advanced OOP: Inheritance, Polymorphism, Abstract Classes, and Interfaces

What is Inheritance?

Inheritance is a key component of OOP that allows us to create a new class from an existing class. The new class that is created is known as the subclass or child class and the existing class from which the child class is created is known as superclass or parent class. A subclass will inherit all the public member data and methods of its parent class, and additional properties unique to the subclass may be added. Private methods of a superclass are not callable from the subclass. Note that all Java classes automatically inherit from the Object class.


Is-A and Has-A Relationships

A subclass is defined as having an is-a relationship with its superclass. In Java, a class can only extend one superclass, but more than one subclass can be derived from one superclass. A has-a relationship means a class has a data field.


Overriding

When a method in the subclass has the same name as a method in the parent class, the method in the subclass can override the method in the superclass by redefining the method (reworking what it does).


Super Keyword

This is used to call the method of the parent class from the child class. If a superclass constructor is used in the subclass, Java differentiates between the different constructors based on the number of parameters.


This Keyword

This is used to refer to the current class. If there is ambiguity between classes, the keyword can help do differentiate between class attributes or parameters with the same name.
Uses:

  1. As a constructor:
    • Used if a class has 2 or more constructors
    • Works much like super keyword
    • Ex. this()
  2. As a variable:
    • Used when a variable name is shared across classes (ex. x or y)
    • Ex. this.variableName
  3. As a method:
    • Used when a method name is shared across classes
    • Ex. this.methodName()
  4. As a parameter:
    • Used to passing a reference to the current object
    • Ex. method(this, arguments)


Instance vs. Local Variables

Instance Variables:

  • Declared inside the class but outside any methods
  • New instances of these variables are created every time a new object is created from the class
  • Every instance of a class will have the same types of variables contained within it, but what's stored in those variables will be different from one object to another

Local Variables:

  • found within a method and nowhere else
  • Only visible within the method

Instance variables are hidden from other classes (even with inheritance). This is called encapsulation.


Set/Get Methods

Set (accessor) methods allow another class to view data stored in an instance variable. Get (mutator) methods allow another class to change data stored in an instance variable.


Rules with Inheritance

  • If the superclass constructor is used it must be the first line in the constructor
  • No argument (default) constructor must always be defined in the superclass to avoid errors
  • Instance variables are to always be declared as private so that they are not accessible by the subclass directly. To access or change them, one needs to use set/get methods
